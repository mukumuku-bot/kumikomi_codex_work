# kumikomi_codex1

スマホのマイク入力を使って、スマホを動かさずに大雑把な音源方向を推定するWebアプリです。

## 仕組み

一般的なスマホの単一マイクだけでは、止まったまま音源方向を推定することはできません。このアプリでは、ブラウザがステレオまたは複数チャンネルのマイク入力を渡してくれる場合に、左右チャンネルの音量差から大雑把に「左方向」「正面付近」「右方向」を表示します。

ブラウザがスマホの複数マイクを単一チャンネルにまとめる場合は、物理的に方向推定ができないため「方向不明」と表示します。

## 使い方

1. ローカルサーバーでこのフォルダを配信します。
2. スマホで同じネットワーク上のURLを開きます。
3. 「測定開始」を押して、マイクの使用を許可します。
4. スマホを動かさずに音源を鳴らします。

## 起動例

```powershell
python -m http.server 8000
```

PCで開く場合:

```text
http://localhost:8000
```

スマホで開く場合は、PCのIPアドレスを使って次のようにアクセスします。

```text
http://PCのIPアドレス:8000
```

ブラウザの仕様により、マイクは `localhost` または HTTPS でないと使えない場合があります。スマホ実機では HTTPS 配信、または開発用トンネルの利用が安定します。

## 実装内容

- Web Audio APIでマイク音量を取得
- ChannelSplitterで左右チャンネルの音量差を計算
- 大雑把に左方向・正面付近・右方向を表示
- 単一チャンネル入力では方向不明を表示
- オフライン表示用の簡易Service Worker
